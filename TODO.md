## Pending / Future Ideas
- [ ] **Optional strict arg-name check for custom `valueof` evaluators (low priority)**: a custom `valueof=CRC32(Type, Data)` passes its referenced fields positionally as `ctx.Args`; transposing the arg names yields a different-but-self-consistent result (encode and decode use the same order) with no compile-time or run-time check. Consider an opt-in mode where an evaluator can assert the arg names it expects (e.g. via `ValueOfContext.Args[i].Name`), catching transposition. Inherent to the positional design; surfaced by the 0.3.2 clean-agent eval as low-severity.
- [ ] **Batch contiguous fixed-width struct fields (perf, both codegen & runtime)**: a struct of N fixed-width scalar fields emits N `w.Write`/`io.ReadFull` calls, and codegen's `var tmp [8]byte` escapes through `w.Write` (≈1 alloc per generated method call that writes a scalar — multiplies under nesting). Batching runs of contiguous fixed-width fields into one buffer + single `Write`/`ReadFull` (like the array bulk path, and like `encoding/binary`'s fixed-size path) removes both. Modest for flat structs; the generator change must track runs across mixed field types. (Surfaced by the 0.3.2 codegen profiling pass.)
- [ ] **Codegen multidimensional arrays over non-scalar leaves (deferred)**: codegen now generates scalar-leaf multidimensional arrays; string/nested-struct/pointer leaves and mixed fixed-array/slice nesting still fail loud (runtime fallback). Supporting them needs the leaf emitter to handle non-scalar element types inside the nested loops. Low priority.
- [ ] **Codegen error: name the ordered parent for a nested type with no own byte order (error legibility, low priority)**: the standalone generator processes each `-type` in isolation, so a nested type that inherits its order from a referencing parent at runtime (e.g. `Record` used only as a field of `Container{_ endian=big}`) fails with `no byte order — declare it on the struct ... or pass -endian`. The message is already actionable (the 0.3.3 clean-agent eval resolved it in seconds via `-endian big`), but it could additionally name the parent ("Record is used by Container(endian=big); pass -endian big or declare it on Record"). Inherent to per-type codegen; a nice-to-have, not a correctness gap. (Surfaced by the 0.3.3 clean-agent eval.)
- [ ] **Codegen custom `valueof` over nested-struct args (Strategy A, deferred)**: the only remaining unsupported arg shape. Would need a fully-static emit of the nested struct into a scratch buffer (retargeting the field-write emitter, plus resolving the nested struct's byte order) instead of the current `ms.MarshalAs` reuse, which can't express a nested struct's order in a standalone tag. Low priority — the runtime interpreter handles it.
